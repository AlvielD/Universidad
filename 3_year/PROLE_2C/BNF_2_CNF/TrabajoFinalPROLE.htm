LA FORMA NORMAL DE CHOMSKY

 

Se dice que una gramática libre de contexto está en Forma Normal de Chomsky si todas sus reglas son de la forma A → a (parte derecha con un único símbolo terminal) o A → B C (parte derecha con dos símbolos no terminales). Si el lenguaje incluye la cadena vacía, entonces se incluye la regla S → λ (siendo S el símbolo inicial).

Toda gramática libre de contexto se puede transformar en una gramática equivalente en Forma Normal de Chomsky mediante el siguiente algoritmo:

  • Eliminar las reglas A → λ y para cada regla en la que aparezca el símbolo A se generan dos reglas, una con A y otra sin A.

  • Para cada símbolo terminal a que aparezca en alguna regla acompañado de más símbolos se crea un nuevo símbolo no terminal A, se añade la regla A → a y se sustituyen las apariciones del terminal a por el no terminal A.

  • Sustituir las reglas de tipo A → B (es decir, las reglas con un único símbolo no terminal en la parte derecha). Para ello, por cada regla B → a se crea una nueva regla A → a y por cada regla  B → C D E  se crea una regla A → C D E.

  • Por cada regla con más de dos símbolos no terminales a la derecha, A → B C D ..., se crean nuevos símbolos terminales N1, N2, ... y  se sustituye la regla por A → B N1,  N1 → C N2, N2 → D...

Ejemplo:

  • Gramática original

  • Primer paso: eliminar reglas lambda (solo afecta al símbolo Args)

  • Segundo paso: crear reglas para los símbolos terminales (crea 8 símbolos y reglas nuevas)

  • Tercer paso: expandir las reglas con un único símbolo

  • Cuarto paso: trocear las reglas con más de dos símbolos

 

ESPECIFICACIÓN LÉXICA

 

La especificación léxica de los ficheros de entrada del trabajo a desarrollar (que describirán gramáticas en notación BNF) está formada por las siguientes categorías:

 Especificación  Expresión regular
 blanco   ( " " | "\r" | "\n" | "\t" )
 comentario "/*" ( ("*")* ~["*", "/"] | "/" )* ("*")+ "/"
 NOTERMINAL ["_","a"-"z","A"-"z"]  ( ["_","a"-"z","A"-"Z","0"-"9"] )*
 TERMIINAL "<" ["_","a"-"z","A"-"z"]  ( ["_","a"-"z","A"-"Z","0"-"9"] )* ">"
 EQ "::="
 BAR   "|"
 SEMICOLON ";"

La categoría NOTERMINAL está asociada a identificadores y se utiliza para referenciar a los símbolos no terminales de la gramática. La categoría TERMINAL se refiere a los símbolos terminales de la gramática, que se escriben como identificadores entre los signos menor ("<") y mayor (">"). La categoría EQ se refiere a la flecha que separa las partes izquierda y derecha de una regla. La categoría BAR reconoce el separador entre diferentes reglas de un mismo símbolo no terminal. La categoría SEMICOLON reconoce el punto y coma que se utiliza como finalizador de las reglas de un símbolo no terminal.

Los blancos se refieren a espacios, tabuladores y saltos de línea y corresponden a componentes léxicos que deben ser filtrados. Los comentarios, que también deben ser filtrados, son iguales que los comentarios multilínea de C.

 

 

ESPECIFICACIÓN SINTÁCTICA

 

La especificación sintáctica de los ficheros de entrada del trabajo a desarrollar (gramáticas en notación BNF)  es la siguiente:

  • Gramatica ::= (  Definicion  )*

  • Definicion  ::=    NOTERMINAL   EQ  ListaReglas  SEMICOLON

  • ListaReglas ::=  Regla  ( BAR  Regla )*

  • Regla ::=  ( NOTERMINAL  |  TERMINAL )*

Una gramática es un conjunto de definiciones. Cada definición describe el conjunto de reglas asociadas a un símbolo no terminal de la gramática. Estas definiciones están formadas por el identificador del símbolo no terminal (NOTERMINAL), seguido del símbolo EQ, seguido de las reglas asociadas a ese símbolo no terminal y terminado en SEMICOLON (es decir, un punto y coma). La lista de reglas está formada por reglas separadas por BAR (es, decir, una barra). Las reglas están formadas por una secuencia de símbolos terminales o no terminales. Para representar una regla lambda se utiliza una secuencia vacía.

 

EJEMPLO DE FUNCIONAMIENTO

 

El siguiente texto describe un ejemplo de entrada correcta para la aplicación a desarrollar. Se trata de la descripción de las expresiones aritméticas formadas por sumas, restas, multiplicaciones y divisiones de números y llamadas a funciones que hemos utilizado como ejemplo del algoritmo de transformación.

  /* Definición de una expresión aritmética como una suma o resta de términos */
  Expr ::= Term
         | Expr <PLUS> Term
         | Expr <MINUS> Term
         ;

  /* Los términos son productos o divisiones de factores */
  Term ::= Factor
         | Term <PROD> Factor
         | Term <DIV> Factor
         ;

  /* Los factores son constantes, expresiones entre paréntesis o llamadas a funciones */
  Factor ::= <NUM>
         | <LPAREN> Expr <RPAREN>
         | <IDENTIFIER> <LPAREN> Args <RPAREN>
         ;

  /* Las reglas lambda se crean con secuencias vacías, como la segunda línea de esta definición */
  Args ::= ArgumentList
         |
         ;

  /* Los argumentos son una lista de expresiones separadas por coma */
  ArgumentList ::= Expr
         | ArgumentList <COMMA> Expr
         ;

 

La aplicación a desarrollar deberá analizar esta gramática y generar la gramática equivalente en Forma Normal de Chomsky utilizando el mismo formato para describir la salida. DESARROLLO DEL TRABAJO

 

Los pasos a realizar en el desarrollo del trabajo son los siguientes:

  1. Desarrollar el fichero JavaCC que genere el analizador léxico y sintáctico que permita procesar los ficheros de entrada.
  2. Definir las clases necesarias para describir el Árbol de Sintaxis Abstracta a utilizar en el trabajo.
  3. Ampliar el analizador sintáctico para considerar la semántica, es decir, construir un analizador semántico que tome como entrada la descripción en modo texto de una Gramática BNF y genere como salida una estructura de datos que represente la Gramática BNF reconocida.
  4. Programar el algoritmo de transformación de la gramática en Forma Normal de Chomsky.
  5. Programar la generación del fichero de salida con el formato planteado.
  6. Realizar un conjunto de pruebas sobre la aplicación.
